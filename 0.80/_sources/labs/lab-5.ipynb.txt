    "For this lab, we will be making new customer segments.  To simplify the process, we will \n",
    "set our configuration to use common reference prices.  This setup puts the control over\n",
    "relative willingness-to-pay between segments into the `ChoiceModel` configs, instead of \n",
    "having it in the `Demand` configs.  This is less flexible (all demands sharing a choice \n",
    "model must have the same reference price multiplier) but also easier to maintain (as \n",
    "there is only one value to look at, in the choice model, for scaling the reference prices,\n",
    "as opposed to unique values for each demand)."
